= Local Development Setup
:navtitle: Local Setup

This guide explains how to get the GDG OIST project running locally.

== Prerequisites
* Node.js (v18+)
* npm (v9+) or bun

== 1. Installation
Clone the project and install dependencies:

[source,bash]
----
cd gdg-oist-crafted
npm install
----

== 2. Environment Configuration
The project requires environment variables to connect to the backend. Create a `.env` file in the project root:

----
VITE_SUPABASE_URL=https://<your-project>.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=eyJhbGciOiJIUzI1NiIsIn...
VITE_SUPABASE_PROJECT_ID=<your-project-id>
----

WARNING: Do not commit your `.env` file! It should remain ignored by Git.

NOTE: When using Lovable Cloud, these variables are automatically configured.

== 3. Running the Development Server
[source,bash]
----
npm run dev
----

The development server (Vite) will boot with Hot Module Replacement (HMR) at `http://localhost:5173/` or `http://localhost:8080/`.

== 4. Building for Production
[source,bash]
----
npm run build
----

Produces an optimized build directory with minified HTML, CSS, and JS.

== 5. Running Tests
[source,bash]
----
npm test
----

Tests are configured with Vitest (`vitest.config.ts`).

== 6. Linting
[source,bash]
----
npm run lint
----

ESLint is configured via `eslint.config.js`.

== 7. Generating Documentation
The project uses Antora for documentation. To build the docs site:

[source,bash]
----
npx antora antora-playbook.yml
----

This generates a static documentation site from the AsciiDoc files in the `docs/` directory.

== Deployment
The project can be deployed to:

* *Lovable* — Click "Publish" in the Lovable editor
* *Vercel* — Connect GitHub repo, auto-deploys on push
* *Netlify* — Similar to Vercel
* *GitHub Pages* — Using the build output

NOTE: Backend (edge functions, database) deploys automatically regardless of frontend deployment method.
